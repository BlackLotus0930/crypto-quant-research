# 资金费套利(cash-and-carry)方案

> 现货 + 反向永续对冲掉价格,**净收资金费(funding)**。市场中性、不预测涨跌、跨 regime 稳健的结构性溢价收割。
> 回测(含死币、扣成本、防泄漏,OOS 2023-2026):**unlevered +15%/yr / Sharpe ~4 / 最大回撤 −4% / ~38 有效币**;加 2-3x 杠杆 → **+31-46%/yr @ −8~13% 回撤**。

---

## 0. 这是什么(一句话)

每个币:仓位方向 = `sign(funding)`,做多/空现货 + 反向永续把**价格对冲掉**,只留下资金费现金流。funding>0(多头拥挤)→ 多现货空永续,收多头付的费;funding<0(空头拥挤)→ 多永续空现货,收空头付的费。横截面市场中性、双侧通吃。

---

## 1. 核心思路:为什么这个能赚,而信息 alpha 不能

- **之前撞的墙(Grossman-Stiglitz)**:从 OHLCV 预测涨跌的横截面信息 alpha,IC 真(~0.025)但**薄**,被竞争压到≈成本线,5bps taker 一吃就平。能从公开价格读出的信息,必然被竞争到接近成本。这是理论,不是我们做得差。
- **换游戏:不挖信息,收风险溢价**。永续资金费是杠杆方为了持仓**付的费**——它是**风险补偿 / 拥挤度的价格**,不是信息。它不会被竞争到零(只会压缩),因为加杠杆的需求是结构性的(总有人要杠杆多/空)。
- **关键区别**:carry 是**慢的、近确定的现金流**(决策时你已知当前 funding),不是快的、噪声大的预测。所以它:
  - **成本不敏感**(换手低 ~26/yr,30bps 重 taker 仍 +18%/yr);
  - **扛 G-S**(收的是风险补偿,不是会被套利掉的信息)。

这是和我们一开始那条路**本质不同**的钱。

---

## 2. 机制(数学)

单个币、单位资本、每 bar:

- 正侧(funding>0):**多 1 现货 + 空 1 永续**。P&L = `spot_ret − perp_ret + funding/8`。
- 负侧(funding<0):**多 1 永续 + 空 1 现货(借币)**。P&L = `perp_ret − spot_ret + (−funding)/8`。
- 统一:`P&L = s·(spot_ret − perp_ret) + s·funding/8`,`s = sign(funding)`。

其中:
- `s·(spot_ret − perp_ret)` = **价格腿** ≈ `−Δbasis`(basis = perp/spot−1)。有界、均值回归 → 累计是小噪声(OOS ~ −4%/yr),**价格被对冲掉了**。
- `s·funding/8` = **funding 腿** = 引擎(8h 费率摊到每小时;OOS funding 腿是收益主体)。
- **双侧 = regime 对称**:牛市多头拥挤(正侧厚)、熊市空头拥挤(负侧厚)。2024 正 funding 占 88%、2026 负 funding 占 47% → 两边都做才不挑 regime。

## 3. 构造设计(CONTROL)

### 为什么 CONTROL(控集中)而不是 keep(任其集中)
利润天然集中在少数 **fat-funding** 币(如 LEVER:sustained ~44%/yr funding)。不控的话:单一正向 book 69% 利润来自一个币,**即便不加杠杆也 −17% 回撤**,且无法安全加杠杆。控制后:分散 → 低回撤 → 可安全加杠杆 → **净收益反而更高**(control L3 +44%@−7% 碾压 keep L1 +24%@−17%)。control 不是丢掉 fat 名字,是**收它但限额**,让任何单名都爆不沉船。

### 具体参数
| 旋钮 | 取值 | 理由 |
| --- | --- | --- |
| 宇宙 | point-in-time top 流动(正侧 top-100、负侧 **top-50 可借**),含死币 | glitch 防线 + 容量诚实 + 负侧只在可借币 |
| 方向/权重 | `sign(funding)`,权重 ∝ \|funding\| | 收得多权重大 |
| **每币帽** | **5%** | 风控:单名爆不沉船(LEVER 即便 2% 帽仍占 36% = 溢价本性集中) |
| 仓位平滑 | EMA hl=120 | carry 慢、低换手(~26/yr)→ 成本不敏感 |
| 调仓 | 每 24bar、rebal 0.3、no-trade band 0.01 | 进一步压换手 |
| **降回撤①** | **basis-momentum 入场过滤**(不进基差正逆向飙=正被挤的币) | 躲逼空,主力降回撤 |
| **降回撤②** | **vol-targeting**(湍流期缩 gross) | 平滑暴露 |
| 杠杆 | 2-3x | 诚实线性(对冲+分散+帽 → 不爆) |

降回撤效果:maxDD −6.9% → **−3.5%**、Sharpe 3.6 → 4.7、年化几乎不伤、薄年(2024)大改善。**机械规则就够,没用模型。**

## 4. 数据与口径(防自欺,rule 0)

- **来源**:Binance Vision(免费、最干净)。永续 1h klines、fundingRate、premiumIndexKlines(基差)、现货 1h klines。
- **含死币**:`download_binance.py --all`(732 币,含已退市)→ 修幸存者偏差。负侧/现货:`--market spot --symbols_npz`。
- **面板/张量**:`build_tensor_pit.py`(point-in-time top-200 流动 + 时长过滤,N=272、含长寿死币)。funding/basis/spot 各自 `build_funding_pit / build_basis_pit / build_spot_pit` 对齐张量 `dates×slots`,merge_asof 因果前向填。
- **rule 0**:① 无幸存者(死币在内、退市尾部计入);② 无前视(funding/basis 用 ≤t、收益用次 bar、funding 向前 accrue);③ 无 peeking(参数 val 前 40% 选、报后 60% OOS);④ 清洗用**流动性截顶**(非 basis 阈值——后者会误删高 funding 溢价源),价格腿只轻 winsor 且报阈值敏感性。

## 5. 验证结果(OOS 2023-2026,含死币,扣双腿成本)

**整套(CONTROL 两侧 + 机械降回撤,负侧 top-50)**:

| 杠杆 | 年化 | Sharpe | 最大回撤 |
| --- | --- | --- | --- |
| 1x | +15% | ~4.0 | −4% |
| 2x | +31% | ~4.0 | −8% |
| 3x | +46% | ~4.0 | −13% |

**逐年(unlevered)**:全正,含 2024(薄,+3~8%)和 2026(+30%+,靠负侧)。

**防自欺检查全过**:
- 成本:总 30bps(重 taker)仍 +18%/yr(换手低 → 成本打不动)。
- 清洗:价格腿 winsor 0.05→不限,OOS 几乎不变(非清洗操纵)。
- 容量:top-30~all 各级全正、2026 全正。
- **集中度**:有效币数 38(双侧),top-1 仅 10%,jackknife 去 top-5 几乎不动(对比单正侧:有效币 3、去 LEVER 即崩)。
- **借币可得性**:负侧限 top-50 可借仍 +14.5%/Sh3.6,83% 负侧利润在 top-30 可借币内。
- **杠杆**:诚实强平模型(无免费止损红利)下 Sharpe 随 L 持平 = 线性、不爆。

## 6. 诚实的风险与天花板

- **Sharpe ~4 偏乐观**:含 funding"近确定 accrual"的性质;实盘扣执行/结算时点会低一截,**保守按 ~2.5-3 看**。这是唯一 backtest 永远证明不了、只能靠前向/实盘验证的点。
- **负侧借币可得性**:空现货需借币,只 liquid 币稳定可借(已用 top-50 限制缓解,但极端行情借券可能收紧)。
- **premium 市场定 + regime 依赖**:funding 收入随杠杆需求暴涨暴落(2024 薄、2025 厚)→ 收益块状,非稳定 15%/yr。
- **容量随 AUM 压缩**:大资金推不动那么多 funding、被迫下沉流动性。
- **仍部分集中**:正侧 ~7-8 有效币、靠少数 fat-funding 名字;双侧分散到 38,但依赖这类名字持续出现。
