## 0. 不变的尺子（先固定度量，再谈改进）

所有实验用同一把尺子，否则数字不可比。


| 度量                       | 定义                                             | 为什么                                                     |
| ------------------------ | ---------------------------------------------- | ------------------------------------------------------- |
| **holdout IC**（主判据）      | 每日横截面 `corr(预测信号, 次日真实收益)`，跨天平均                | 直接对齐「按横截面方向排序」这件能变现的事；CRPS 对它近乎失明                       |
| **IR**（年化）               | IC 的年化信息比 = mean(IC)/std(IC)·√(252×bars/天)     | 看信号稳不稳；**日内按 bar 年化**（日频 bars/天=1）。修了"日内被√252低估√7"的 bug |
| **输入置零 ablation**        | 把价量输入全置 0（只留 mask+dt）再算 IC                     | 防自欺：若置零后 IC≈真实，说明模型没在用价量（蹭了泄漏/常数）                       |
| **holdout CRPS skill**   | `1 − CRPS_model / CRPS_RW`（辅判据）                | 分布质量，用于仓位 sizing；不是赚钱的主证据                               |
| **净 Sharpe@2bps**（GO 门槛） | 风险平价 book（仓位∝信号/vol）扣 **2bps** 单边成本后的年化 Sharpe | 钱的终判。同时报 2/5/10bps 看敏感性                                 |


**GO 门槛**：净 Sharpe@**2bps** ≥ 0.5。**成本从 5→2bps**：我们 universe 是 top 流动大盘股（中位日成交额 $135M+），小资金 book 真实成本=半价差+冲击 ~1-2bps，5bps 对大盘股偏苛刻。
**执行点(rebal+band)和 horizon Δt 全在 val 上按成本选，holdout 只用来报**（`best_exec_at`+val 选 horizon；不在 holdout 上扫/挑=不过拟合）。最优 rebal 强依赖成本：0bp→1.0、1bp→0.7、2bp→0.5、5bp→0.1。
**切分加 embargo**：标签窗末端(a+Δt_max)跨进下一段的 anchor 一律剔除（防边界标签前视；train/val/holdout 跨段数=0 已验）。
**🟡 有信号未变现**：best IR ≥ 1.0 / 扣 2bps 净正但 <0.5 → 增厚信号（频率/规模），不是 NO-GO。
**❌ NO-GO**：IC≈0 且不赚。

切分（防泄漏，按时间，holdout 只碰 1–2 次）：


| 桶       | 区间                      | 用途       |
| ------- | ----------------------- | -------- |
| train   | 2012-01-01 → 2020-12-31 | 训练       |
| val     | 2020-12-31 → 2022-12-31 | 选模型 / 早停 |
| holdout | 2022-12-31 → 今          | 终判，最少触碰  |

### 🧱 元结论(三连证,极稳)

- **更深一层怀疑**:问题可能一半在**策略框架**(加密 beta 太强,横截面长短锁天生难做干净),不只是信号本身。
- 方法论收获:Dune→信号→纪律探针 管线跑通且便宜(25cr),要继续探更冷门链上信号很快。

# 📗 阶段二(E21–E31):carry 诞生与打磨

## E21 — funding carry 结构性溢价(换游戏:不挖信息、收风险溢价)

**背景**:E18 确认 OHLCV-横截面-信息alpha 撞 Grossman-Stiglitz 墙(真但薄、近期过不了成本);beta 中性测试证明 beta 非真凶(对冲后全程 Sharpe +0.45→+0.47 不变)。**换游戏** → funding carry:永续资金费=结构性溢价(杠杆多头付费给空头,加杠杆需求结构性、不被竞争到零)。干净宇宙(含死币 N=272)funding 均值 **+6.3%/yr**(longs 付费,方向确认,比 survivor-only 的 3.8% 更高=死/alt 币 funding 更热)。

**做法**:横截面市场中性,空高funding/多低funding(信号=当前 funding 横截面 zscore,≤t 因果),含死币、扣成本、val 选 hl 报后 60%、`carry_backtest.py`。

**结果(后 60% OOS,逐年 net@5)**:


| 区间       | net@5     |
| -------- | --------- |
| 全程       | +0.34     |
| 2023     | +1.22     |
| 2024     | +0.14     |
| 2025     | +0.42     |
| **2026** | **−1.53** |


**两腿分解(关键洞察)**:


| 腿          | Sharpe    | 年化         | 累计     |
| ---------- | --------- | ---------- | ------ |
| funding 收割 | **+54.8** | **+69.0%** | +2.656 |
| 价格         | −1.28     | −51.9%     | −1.998 |
| 合计毛        |           |            | +0.658 |


- **funding 腿真实、巨大、近确定性 accrual**(Sharpe 54 = 准无风险收取,不是噪声 alpha)。
- **价格腿是杀手**:空"拥挤多头"=吃它们的价格上涨(本质=短动量),趋势市被碾(2026 价格腿崩 → net −1.53)。funding(+2.656)勉强压过价格(−1.998)。

**容量探针(net@5 OOS)**:top-50 **−0.68** / top-100 **−0.30** / 全272 +0.34。**可交易的流动版是负的;溢价只活在不流动的山寨长尾。**

**退市尾部**:短腿在退市币上 **+0.561**(空将死币赚钱,非逼空)→ 尾部不是杀手。

- **判定 🔴(裸 perp-carry)**:net@5 +0.34<0.8、2026 −1.53、容量负 → **不进 Phase3**(模型叠加在坏底座=抛光废铁)。也在衰减(2020-23 极好→2026 死,同 recency 病)。
- **但溢价本身是 🟢**:funding +69%/yr 真实。**杀手是价格腿**(空拥挤多头的价格风险),不是 funding。
- **结果精确指出修法 = 去掉价格风险**。结构正解 = **cash-and-carry(多现货 + 空永续,同币)**:价格被现货对冲、净收 funding。我们**没有现货数据**;perp-only 长短腿是不同币(基差风险)=劣质代理。
- **下一步岔路**:(A) 下现货数据 → 真 cash-and-carry(结构去价格风险,需真实摩擦:基差/借币/funding翻负/逼空);(B) 模型当价格腿过滤器(反转预测哪些高funding币要反转=可安全做空)。容量在流动端受限是真约束。

## E22 — 真 cash-and-carry(价格中性 funding 收割):**第一个干净 🟢**

**背景**:E21 分解出 funding 腿 +69%/yr 真、价格腿是杀手。cash-and-carry(多现货+空永续,**同币**)按构造对冲掉价格腿。用 premiumIndex(基差,`--all` 含死币下载)算价格腿 P&L=`−Δbasis`,**无需现货数据**即可先验证(`cashcarry_backtest.py`)。

**数据**:`funding_pit`(272/272,+6.3%/yr)+ `basis_pit`(272/272,premiumIndex)。每单位每bar P&L=`funding/8 − Δbasis`。只在 funding>0 币上(空永续收)、funding 权重、价格中性、**双腿成本**、含死币、慢平滑 hl=72(turnover 4/yr)。

**结果(后60% OOS,net@5,双腿成本)**:


| 区间       | net@5     | 年化                            |
| -------- | --------- | ----------------------------- |
| OOS全     | **+4.15** | +16.2%                        |
| 2023     | −0.45     | −1.3%(post-FTX 低funding熊尾,≈平) |
| 2024     | +3.14     | +7.0%                         |
| 2025     | +8.90     | +54.5%                        |
| **2026** | **+3.93** | +12.1%                        |


**两腿分解**:funding收割 Sharpe 14.8 / +18%/yr(引擎);基差腿 −1.5%/yr(有界噪声,确认价格中性)。

**成本不敏感(杀手级特征)**:net@2 +4.21 / net@10 +4.05 —— turnover 仅 4/yr,成本几乎不动它。和所有前策略(5bps 就死)**相反**:它是**慢的结构性溢价、不是快的信息边**,所以过 G-S 墙。

**容量(net@5 OOS)**:top-20 +1.13 / top-50 +1.91 / top-100 **+4.73** / 全 +4.15。**top-100 最优**(非全靠不流动垃圾),可真实规模;只 top-20 主流 2026 −1.04(主流 funding 被挤压)。回撤 −0.066(极小)。

**2026 是 turnover/成本问题(已证)**:hl 扫描 0→72 单调修好 2026(−1.22→+3.93),turnover 25→4。

- **判定 🟢(第一个干净的)**:net@5 +4.15、逐年≈正(含 2026 +3.93)、容量好、成本不敏感、回撤小、含死币无前视。
- **但 Sharpe 4.15 偏乐观(诚实)**:premiumIndex 近似、**未建模**(a)空永续腿**逼空强平**(真尾部;我假设持有穿过基差均值回归,但强平=永久损失)(b)双腿真实滑点/借币/保证金(c)funding 结算时点。真实可交易夏普更低(真 basis 基金扣摩擦后 ~1-2),但**仍是真实、市场中性、能赚钱、且非 G-S 墙**的策略。
- **下一步去乐观化**:① 下现货 → 真实 hedged P&L 含基差爆裂 + 保证金/强平模型(替代 premiumIndex 近似);② 真实双腿滑点;③ **模型叠加(Phase3)用分布预测躲将逼空的币 + 按尾部定仓** → 砍强平尾部(模型在"已能赚"的策略上才真正发挥价值)。

## E33 — 借币问题的解:期限结构(死路)→ 跨所是负侧唯一零借币通路(数据钉死)

**问题(E20g 之后发现)**:实盘真实 Binance 借率(median 46%/yr,肥币 60–130%)→ 16 个负侧币里 9 个 borrow > funding → 单所"短现货收负 funding"净 ≈ 0。**洞察:funding 越负 ⇔ 借币越贵,是同一个拥挤的两面**(G-S:均匀拥挤那部分负 funding 不是免费午餐,借率正确定价它)。

**思路:借币只因选了"现货"当对冲。换衍生品对冲 long-perp → 零借币。** 三种对冲腿:① 短现货(负侧要借,贵)② 短另一所同币永续(跨所,零借币,收价差)③ 短同所季度合约(期限结构,零借币)。

**调查 ③ 期限结构(perp vs 季度)= 死路(本轮新查证):**

- Binance 季度交割合约**只有** BTC/ETH(U本位 2 个)+ BNB/SOL/XRP(币本位)。
- 这些都是正 funding 大盘(BTC +12%、ETH +14% 年化,仅 14% 时间为负且 min −0.3%/8h 极小)。
- **年化 mean funding < −2% 的 49 个借币问题币,有季度合约的 = 0 个。** 季度合约只存在于"不缺负 funding 可收"的币上;真负 funding 的小 alt 一个都没有。→ **期限结构解不了借币问题。**

**调查 ② 跨所 = 负侧唯一可行的零借币通路(数据钉死):**

- 49 个负 funding 币里 **42 个在 Bybit 上有**(覆盖 >30%)。
- 跨所收割 = |f_bin − f_byb| 价差(零借币)。诚实可交易测(**滞后已知 funding 选所**,非完美选所):
  - 负 funding 币跨所收割 **中位 +11.5%/yr、均值 +28%**(funding 腿,未扣价格腿/成本);
  - **留存率 84%**(滞后/完美上界)→ 价差**持久,非噪声**;**100% 的负侧币收益为正**。
- 极端币(AKRO −367%、SXP −87%、DODO −78%)的跨所价差 ≈ 整个 Binance 负 funding → 拥挤是 **Binance 特有 venue 错位**(Bybit 没那么负),不是均匀拥挤 → 跨所能干净捞走。与 E32 `xvenue_net` 真实净额(OOS +10.4%/yr、价格腿 haircut 仅 −0.3%)一致。

**结论(策略结构定型,全程衍生品对冲、零借币):**

1. **正 funding → long 现货 + short 永续**(cash-and-carry,无借币)= 干净主干。
2. **负 funding → 跨所 perp-perp**(零借币,中位 +11.5%/yr,持久,100% 正)= 负侧解。
3. **彻底砍掉"单所短现货负侧"**(借币杀死),只留少数 funding 远超借率的极端币。
4. **期限结构 = 死路**,封存(小 alt 无季度合约)。

→ 借币问题不是"打败借率"(那是真实成本),而是**把负侧 harvest 路由到跨所 perp-perp**,从结构上绕开现货借币。脚本:`research/carry_audit_numbers.py`(借率核算)、本轮即席分析(季度合约可用性 + 跨所负侧覆盖/收割)。

**④ 反应窗口(止损来得及吗,分辨率递进)**:1h=98%同bar击穿(假象,bar太粗)→ 15min=40%(2025-10)→ **1min:中位 2分钟窗口、仅 5% 真瞬时 gap、96% 可救**(实时拉 Binance 1m 验证)。**最细数据**:历史 tick(aggTrades)、实盘 WebSocket 亚秒;2分钟窗口绰绰有余。残留=单 tick 大单跳,任何分辨率不归零。

**⑤ 为什么跨所顶在 3x、5x 不行(数据钉死)**:不是反应窗口(5x 也够);是**止损线被正常波动磨死**——要扛 5x 强平,止损线设 13%,而山寨**平时每 8 天就动 13%**(15min OHLC 实测:触线币·天 3x=3.5%/28天、5x=12.2%/8天)→ 止损疯狂误触发→ funding 被自己止损磨光。**3x 甜点**:止损 22%、每币 ~28 天才触=只崩盘触发=纯尾部保护。

**⑥ 提前降杠杆能预测崩盘吗(Q2)**:近2周已实现波动→下周 BTC 最坏单日 **Spearman IC −0.46**(强)、funding 拥挤 −0.26。但**伏击型崩盘从平静里炸**:2025-10 前一周波动仅 9 分位、funding 17 分位→预测不到。→ **vol-targeting 躲预告型崩盘(免费纵深),躲不掉伏击型**;反应止损(④)才抓伏击,但有 5% 残留。两法都在"最快+从平静"的崩盘上有洞。

**升级杠杆政策(纠正 E54)**:**carry 6x**(同所,COVID 免疫,约束=funding翻负 maxDD)/ **跨所 3x**(配亚秒 per-pair 止损,夺 96% 崩盘可救;裸=2x)/ **组合 blended ~4.5x**。**5x+ 跨所止损掐死策略,不可行**。vol-targeting 当免费纵深层。**真要提组合收益:加杠杆在 carry(6x),跨所死守 ≤3x。**

**caveat**:Binance(CEX)数据;HL(DEX)链上微结构、崩盘里平仓滑点、执行(平危险腿时另一腿没成交=自造孤儿腿)→ 都要 HL 前向证。
