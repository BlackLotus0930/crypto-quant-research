# 风控:尾部控制 + 对手方分散

> 策略的市场/基差尾部很小(回测组合最差单日 −0.4%、maxDD −3.6%)。**真正会让我们栽的是回测里根本没有的尾部:对手方冻结、强平、稳定币脱锚。** 这页是把这些"没盖的尾部"用结构盖住的规则。配合 `checkup.py`(量市场尾)一起看。

## 0. 核心数学:为什么对手方是第一优先

钱是**乘法**滚的。单一所归零不是"少赚",是**终止复利**:
- 损失 40% → 要 +67% 回本(可恢复)。
- 损失 75% → 要 +300% 回本(基本回不来)。
- 损失 100% → 出局。

所以风控的目标函数不是"少亏一点",是**让任何单一事件都不致命**。这比多挤几个点年化重要得多。

## 1. 现状诊断:我们极度依赖 Gate(真实数据)

当前前向书(`data/paper/state.json`)各 venue 毛敞口:

| venue | 毛敞口(占总资本) | 性质 |
|---|---|---|
| **Gate** | **134%** | 中心化托管(冻结/暴雷/退出风险) |
| HL | 34% | DEX(自托管,链上/桥风险) |

**为什么 Gate 这么高**:carry 腿两条腿(多现货 + 空永续)**全在 Gate**;跨所腿一半也路由到 Gate。→ **Gate 冻结 = carry 全书 + 半个跨所书一起卡死**。这是我们最大的单点风险。

**关键澄清:退出 ≠ 没收(用户更正,对)**
- **监管退出是有序的**:看 Binance(2023)/OKX 退加拿大先例——**停新注册 + 给老用户几个月窗口平仓提现**,不是冻结没收。所以"Gate 退出加拿大"**不是**我们怕的尾部,有充足时间撤。
- **真正怕的对手方尾部是突发的**:暴雷/欺诈/被盗/个人账户被风控冻结——**零预警**。先例:FTX(一夜停提)、QuadrigaCX(加拿大所,创始人死带走私钥 $1.9 亿)、Mt.Gox。Gate 透明度低、监管轻 → 这类突发风险**高于**大所。
- **Gate vs HL 性质不同**:Gate=中心化托管(突发暴雷/冻结);HL=DEX 自托管,你持私钥、可无许可提现(风险是合约/桥/预言机,非"账户被冻")。
- → 结论:**不为"有序退出"换所(没必要);要防的是"突发暴雷/冻结",而防它不靠换所,靠"别在所里放超过工作保证金的钱"。**

## 2. 对手方分散规则(硬约束,不可谈判)

| # | 规则 | 数值 | 作用 |
|---|---|---|---|
| R1 | **冷储备**:留一部分 NAV 在自托管钱包,永不上所 | ≥ 20% NAV | 任何所全损也能重建 |
| R2 | **单一中心化托管帽**:Gate 上的**实际余额/抵押**上限(不换所,靠扫款压低) | ≤ 40% NAV | 突发暴雷最多亏 40%=可恢复 |
| R3 | **扫款纪律**:超出工作保证金的盈利定期提到冷储备 | ≥ 每周一次 | 不让 NAV 在所里堆积 |
| R4 | **提现金丝雀**:每周从每个所试提一笔小额 | 固定小额 | 提不出=最早预警,立即撤该所 |
| R5 | **杠杆闸**:差异化(见 §3)——**跨所腿 ≤2x 硬顶**、carry 腿 ≤6x、组合 blended ≤4x | 跨所2x/carry6x | 跨所是崩盘悬崖源,永不超 2x |
| R6 | **熔断**:所异常(提现失败/脱锚/基差爆裂)→ 立即平该所书 | — | 把损失锁在单所 |

**R2 怎么做到(不换所)**:Gate 134% 是**毛敞口**(名义),但对手方风险看的是 **Gate 实际持有你多少钱**(现货币值 + 永续保证金),不是名义。降它**不用换所**:
1. **扫款**(R3):Gate 只留够维持仓位的工作保证金,盈利和闲置 USDC 定期提到自托管冷钱包 → Gate 实际余额 ≤ 40% NAV,即使突发暴雷也只亏工作保证金。
2. **冷储备**(R1):≥20% NAV 永远在自托管,任何所全损都能重建。
3. (可选,非必须)长大后加第三个所摊薄,自然更易满足 R2。
→ **核心:不是"把币搬出 Gate 换地方交易",是"不在 Gate 留闲钱"。** 现货腿该在 Gate 就在 Gate,但别让 NAV 堆在那。

## 3. 杠杆政策(真实崩盘日实测定;纠正旧版 6x)

> **纠正**:旧版按"合成压力测"给到 6x —— **错了**,合成测忽略了崩盘日的**逐腿强平**机制(一腿被强平→对冲断裂→裸头寸)。
> 权威口径 = 真实年度崩盘日 + 真实强平模型(`research/stress_crash.py` / `stress_protected.py` / `tail_hedge_crash.py`,台账 E55)。

### 真实年度崩盘日:跨所书强平%(裸 uniform,维持保证金 2%)

| 杠杆 | 2021-05-19 中国禁(中位币动35%) | 2022-05-11 LUNA(24%) | 2025-10-10(28%) |
|---|---|---|---|
| 2x | 0% 强平 | 1% | 2% |
| **3x** | **75% → −70%** | **52% → −49%** | **48% → −45%** |
| 5x | 95% → −86% | 93% → −83% | 90% → −81% |
| **6x** | **~98% → −80%+** | ~95% → −83% | ~95% → −81% |

### 三个铁的事实
1. **跨所书 L=3 有强平悬崖**:年度崩盘中位币动 24-35%,而强平距离=1/L−mm(L3≈31%)→ L≥3 群体强平。**L=2 安全**(0-10%)。
2. **6x uniform = 每年归零**:L=5-6 崩盘日 90-95% 强平 → −80%+。这种崩盘 crypto **每年都有**(中国禁/LUNA/FTX/2025-10),不是百年事件 → 6x 一年内必爆。
3. **裸 uniform 这些保护突破不了 2x**:波动分层救不了最惨 2021;BTC put 经济死;BTC 整书熔断抓不到山寨领跌(2025-10)。**但每对亚秒止损能(见下)。**

### 突破口:每对保证金止损(亚秒)→ 跨所可到 3x
亏损腿亏到 **70%×强平距离** 就**把整对一起平**(对冲完整退出,不留孤儿腿)。反应窗口实测(分辨率递进,E55):1h=98%同bar(假象)→ 15min=40% → **1min:中位 2 分钟窗口、仅 5% 真瞬时 gap、96% 可救**。**最细数据**:实盘 WebSocket 亚秒监控,2 分钟窗口绰绰有余 → **止损可行,跨所夺 96% 崩盘强平 → 可上 3x**。残留 5% 单 tick gap 不可消除。

### 为什么 3x 而非 5x(数据钉死,E55)
不是反应窗口(5x 也够),是**止损线被正常波动磨死**:扛 5x 需止损线 13%,而山寨**平时每 8 天就动 13%**(15min 实测:触线币·天 3x=3.5%/28天、5x=12.2%/8天)→ funding 被自己止损磨光。**3x 甜点**:止损 22%、每币 ~28 天才触=只崩盘触发=纯尾部保护。

### carry vs 跨所:不对称(关键)
- **carry(同所 cross-margin)无孤儿腿**:COVID 10x 仍 −14.6%,约束=funding 翻负 maxDD(−10.9%×L:6x=−65%)→ 可上 6x。
- **跨所(跨账户)**:一腿强平→对冲断→孤儿腿→悬崖。**唯一杠杆约束源。**

### 政策:差异化杠杆(不 uniform)

| 腿 | 上限 | 依据 |
|---|---|---|
| **carry** | **6x** | 同所无悬崖;约束 funding 翻负 maxDD −10.9%×L |
| **跨所** | **裸 2x / 配亚秒止损 3x** | L≥4 止损线进正常波动区→磨死;5x+ 不可行 |
| **组合(差异化)** | **~4.5x blended**(carry6x + 跨所3x) | 年度崩盘可恢复 |
| ~~uniform 6x / 跨所 5x+~~ | ❌ **禁止** | 每年崩盘 −80% 归零 / 止损掐死策略 |

### 上线路径(渐进)
1x 前向 → 验证 → **carry 3x / 跨所 2x**(裸,blended ~2.5x)→ **亚秒 per-pair 止损 live + HL 前向证 2分钟窗口/滑点** → **carry 6x / 跨所 3x**(blended ~4.5x)。**跨所永不超 3x**(再高止损磨死);要提收益加在 carry。配 vol-targeting(波动升→预降一档)当免费纵深。

## 4. $5-10k 起步的具体配置

| 桶 | 占 NAV | 放哪 |
|---|---|---|
| 冷储备(R1) | 25% | 自托管钱包 USDC/BTC,不上所 |
| Gate 实际余额(R2 帽) | ≤ 40% | carry 两腿 + 半跨所的工作保证金;**靠扫款压低,不换所** |
| HL | ≤ 35% | 跨所 HL 腿(DEX 自托管,风险性质不同) |

- 杠杆起步 **1x**,扫款每周,金丝雀每周。
- 验证后差异化加杠杆(§3):**carry 腿可加多(无强平悬崖)、跨所腿死守 2x**。
- 长大后:加第三个所(R2 自然更易满足)。

## 6. 自动减仓系统设计(`risk_monitor.py`,台账 E54)

**为什么**:强平是终结态(被迫在最差点平,本该弹回的损失锁死)。自动减仓把"慢回撤"变"可管理"。**注意(E55 纠正)**:它拦得住**慢回撤**(carry 腿的 funding 翻负/基差慢漂),拦**不住跨所书崩盘日的瞬时群体强平**(崩盘太快,1-5分钟轮询来不及)→ **它不能把跨所腿抬过 2x**。它的真价值=保护 carry 腿的慢回撤(让 carry 安心上 6x)+ 组合慢bleed,**近免费保险(平时拖累=0)**。

### 监控什么(输入)
| 信号 | 来源 | 防什么 |
|---|---|---|
| 总 NAV + 历史峰值 → 回撤 | `data/paper/nav.csv` / track_record | 多bar累积回撤 |
| 每所 权益 / 维持保证金(距强平多远) | reconcile **扩展**拉账户状态(HL `clearinghouseState.marginSummary`;Gate/OKX account) | 单腿强平 |
| 每仓 跨所基差(mark_A vs mark_B) | `venues.fetch()` 实时 | 基差爆裂 |
| 每所 健康(提现/API/脱锚) | reconcile + 金丝雀 | 对手方突发 |

### 触发(分层,软→硬;迟滞防抖)
- **L1 回撤闸**:总回撤 > **15%** → 全书 reduce-only 降到 **25%×基础杠杆**;回撤回到 **−6%** 上方恢复。(=autodelev 模拟规则,实测把 6x 危机 −86%→−60%。)
- **L2 保证金闸(每所)**:某所 `权益/维持保证金 < 2.0`(距强平<2倍)→ ① 优先 reduce-only 减该所仓(快)② 或从冷储备补保证金(有转账时延)。
- **L3 基差闸(每仓)**:某仓跨所基差 > **5%** → flatten 该仓(基差爆裂熔断 = R6)。
- **L4 kill-switch**:某所异常(提现失败/API错/价脱锚)→ flatten 该所书 + 停新仓。

### 动作
- **降杠杆**:所有仓按比例 reduce-only(调 `execution.py` 的 reduce-only 目标)。
- **补保证金**:从冷储备转入(转账慢→优先减仓更快)。
- **恢复**:回撤恢复后**逐步**回基础杠杆(不一次拉满)。

### 速度 & 硬限制
- 保证金/基差闸轮询 **1-5 分钟**;kill-switch 事件驱动。
- **拦不住轮询周期内的单bar瞬时gap** → 这就是 ~5-6x 天花板的来源(E54)。别指望它救 10x+。

### 对接现有 + 上线分阶段
1. **扩展 reconcile**:除持仓外,加拉**每所权益+维持保证金率**。
2. **新建 `risk_monitor.py`**:loop 调 reconcile + venues.fetch → 算 L1-L4 → **先 DRY-RUN 只 log "本应减仓"**(`data/exec/risk_log.jsonl`),在 paper 上验证触发正确。
3. 验证后接 `execution.py` reduce-only(小仓 live → 全量)。
- **上线杠杆门**:自动减仓 DRY-RUN 验证 + R1/R3 达标 → 才从 1x 往 3-4x;减仓 live 验证 + 接受 −60% 危机 → 才 5-6x。

---

**一句话**:市场尾部我们已经量过、很小;真尾部不是"Gate 退出加拿大"(有序、有窗口),是"Gate 突发暴雷/冻结"。**防它不靠换所,靠 R1(冷储备≥20%)+ R3(扫款,Gate 只留工作保证金)→ 突发也只亏工作保证金。** 这比任何参数优化都值钱。
