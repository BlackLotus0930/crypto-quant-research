- **价差**:carry 永续 5bps / **现货 16bps**(贵在现货腿);跨所两所永续 ~4bps。**价差 vs funding 相关 +0.40**(肥币更宽)。
- **深度**(@$5k,~$224/币):多数够,**但最肥的薄**——PORTAL 深度 $151 < 单 $224、SPELL $424、HEI/ORCA 几百。**funding 最肥的币簿子最薄 → 滑点**。
- **借券**:公共端点要 auth → **必须账户手查**负侧每币(可借/利率/上限)。
- dry-run(`execution.py`):最小可投 ~$1-2k;全覆盖毛杠杆 1.9x(贴 2x 帽);maker 7.8bps vs taker 12.2bps;负侧借券 ~50% 资本。

## 1. 账户 / 权限 / 安全(先做,最危险的是 key)

- **两所账户**:Binance(现货 + 现货杠杆借币 + U本位永续)、Bybit(U本位永续)。
- **API key**:开 **交易 + 只读,绝不开提现**;**IP 白名单**;key 存环境变量/密钥管理,**不进代码/git**。
- **最小权限**:carry 只需 Binance;跨所需 Bybit。先只开要用的。

## 2. 资金分配(treasury)

- carry 腿(Binance):现货多头要现金、负侧空现货要保证金+借币、永续腿要保证金。**cross-margin**(现货当永续抵押)提资本效率。
- 跨所腿:Binance 永续保证金 + Bybit 永续保证金(两边都要放钱)。
- **起步建议**:先只 carry(单所 Binance,最简),钱都在 Binance;跨所(要 Bybit 放钱、跨所搬钱)**第二阶段**再加。
- 留**保证金缓冲**(别满仓占保证金),应对急行情。

## 3. 借券(负侧硬依赖)——上线前必查

- 负侧 = 空现货 = 借币。当前书 100% 负侧 → **每个负侧币都要能借**。
- 上线前用账户查每币:**① 可不可借(cross/isolated margin 资产)② 借币年化利率(直接减负侧收益)③ 借币上限**。
- **借不到/利率过高的币 → 从负侧剔除**(策略宇宙据此收紧)。这会改变实盘可投的书,要先知道。

## 4. 执行(直接决定净收益——"最好结果"的核心)

- **maker 优先**:挂买一/卖一,赚价差而非付。carry **现货腿 16bps** 最该用 maker(省最多)。慢策略挂得起单。
- **depth-aware sizing(关键改进)**:别只用 5% 帽,**按每币簿子深度限单**(如单 ≤ 10bps 深度的 1/3)。体检显示 PORTAL/SPELL/HEI/ORCA 等肥币簿薄——**对这些币降权或拆单慢做**,否则滑点吃掉肥 funding。
- **腿要一起成交**:防裸腿。先成交不流动的那条腿(现货/小所),再对冲流动的那条。
- **拆单 + 耐心**:小 size 本就不急,分批挂、给时间成交。

## 5. 风控 / 保证金(实盘强制,非回测)

- **杠杆帽 2x**(按 2021 真实 maxDD −8.8% 定;起步可更低 ~1x 再 ramp)。全覆盖已 1.9x → 余量紧。
- **保证金监控**:两所各自维持保证金,留缓冲,逼近阈值**自动降杠杆**(防强平=真实尾部)。
- **kill switch**:数据 stale / 持仓异常偏离目标 / 大额亏损 / 保证金告急 / API 报错 → **拉平或暂停**。
- 下单前 sanity check(别发疯单)。

## 6. 可靠性(24/7)

- crypto 不睡、funding 一天结算 3 次、持仓要持续看 → **跑在稳定 VPS,不是笔记本**。
- 状态持久 + 崩溃自动恢复;API 限流/报错/部分成交 的重试与幂等。

## 7. 监控 / 对账

- 看板:持仓、分腿 P&L、保证金健康、数据新鲜度。
- 告警:裸腿、保证金低、数据断、错误。
- **对账**:实际持仓 vs 目标(execution.py 的 reconcile)、**funding 实收 vs 预期**(前向验证的核心)、手续费、P&L 归因。

## 8. 上线流程(分阶段,稳健压倒一切)

1. **dry-run 对账**(现在):execution.py 读真实持仓(只读 key)→ 打印订单,人工核对几天。
2. **极小额 carry-only**(Binance 单所,最简):真下单、验真实成交/funding 到账/保证金/借券。
3. **加跨所腿**(两所):验跨所同步、Bybit 执行、跨所搬钱。
4. **逐步放大**:信心+数据够了再加 size,起步低杠杆再 ramp。

## "最好结果"的三个杠杆(超出"能跑")

1. **maker 执行**(尤其现货腿 16bps)→ 把成本变接近 0/返佣,薄边策略这几 bps 是真钱。

3. **借券成本管理**(选低借率币、负侧据可借性收紧)→ 负侧净收益 = funding − 借率。
