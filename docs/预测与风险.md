## 2. 风险登记册(会怎么亏)

### 2.1 🔴 Sharpe 虚高 = 漏了风险(最重要的认知)
持续的高 Sharpe(6-9)在真实市场基本不存在。我们的高 Sharpe 来自回测**低估真实方差**:
- funding accrual 近确定理想化(连续确定地记收入,真实是离散结算、会变);
- 小时收盘**低估日内**基差/价格波动与执行滑点;
- 退市币幸存者偏差(只拿到现存币)。
→ **绝对 Sharpe 是"卖保险的保费",没记进偶尔的"赔大的"。只有实盘能定真值。**

### 2.2 🔴 交易所暴雷 / 对手方风险(跨所尤甚)
资金费套利的腿压在交易所上。**交易所暴雷/冻结/跑路 → 该所的腿可能归零**(FTX 教训)。
- 单所 cash-and-carry:两腿都在一个所 → 那个所暴雷 = 该所资金受损。
- 跨所:两腿在两个所 → 一个所暴雷 = 丢一条腿 + 另一条腿变裸单。
- **跨所价差之所以存在、没被套平,部分正是这个风险的报酬**——我们在"卖交易所不出事的保险"。回测 2021-26 的 Binance/Bybit 没暴雷,所以只看到保费、没看到赔付。
- 缓解:**分散多所、单所敞口设上限、闲置资金提走、跨所流仓位压小**。

### 2.3 🟡 funding regime 反转
funding 是杠杆需求的价格,会随 regime 暴涨暴落甚至反转(2026 正侧萎缩、负侧暴涨)。双侧(正+反向 carry)缓解,但平淡期收益薄。

### 2.4 🟡 单名基差爆裂 / 逼空
肥 funding 集中在少数拥挤币;它们可能逼空/基差爆裂。靠 **5% 每币帽 + basis-momentum 入场过滤**控制(尾部可预测,AUC 0.75)。压力测试:含 COVID/LUNA/FTX 逐年正、回撤分散非系统。

### 2.5 🟡 借券可得性(负侧)
反向 carry = 空现货,需借币;极端行情借券收紧/召回。限 top-50 流动缓解(83% 利润在可借币内)。

### 2.6 🟡 杠杆下强平 / 容量
杠杆放大回撤(线性);按真实 maxDD −8.8%(2021)保守用 2x。容量:大资金压 funding(小资金不受限,见 Q1)。

## 4. 铁律(贯穿始终)

1. **回测 Sharpe 越漂亮 → 越要找漏了什么风险**(carry/套利的高 Sharpe 几乎总是 accrual 理想化 + 未采样的尾部)。
2. **可信的是机制与比例,不是绝对数**(corr≈0、组合 1.7×、价格被对冲——这些稳;Sharpe 6/7 不稳)。
3. **真值只能靠实盘/前向**(已在跑);上线后**按"随时可能赔光一条腿"sizing**,绝不因回测漂亮重仓。
4. 全程纪律:含死币、无前视、无 peeking、扣成本、压力 regime、相关尾部、诚实杠杆——这套纪律本身是最大的资产。
