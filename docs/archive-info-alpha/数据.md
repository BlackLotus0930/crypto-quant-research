# 数据 Spec（交易目标：美国市场）

> **数据源已从 akshare/新浪（免费）切到 Massive.com flat files（付费）。** 原因见下。
> E0–E6 实验用的是 akshare 日频（legacy，见 [实验.md](实验.md)）；**从频率验证起，全部走 Massive。**

## 为什么换源（别迷惑：我们曾坚持"只用免费/非北美数据"）

- 原约束是**为了省钱**避开贵的北美数据。
- 但 E0–E6 证明：免费**日频**数据上，**便宜的后处理杠杆已榨干**——广度放大（E5）、因子中性化（E6）都失败，信号真但薄（净 Sharpe@2bps ~+0.1）。
- 唯一剩下的"质变"杠杆是**频率（日内）**，而免费源（akshare/新浪）只有日频。
- 故**故意推翻原约束**，付费拉 Massive（机构级 SIP 行情，Polygon 同类）。**Stocks Developer 档 $79/月。**

## 数据源与约束

- 源：**Massive.com flat files（S3）**。凭证在 `.massive_key`（S3 access key/secret/endpoint/bucket）——**勿入 git**。
- S3：endpoint `https://files.massive.com`，bucket `flatfiles`，前缀 `us_stocks_sip/`。**我们只订 US Stocks → 只能下这个前缀**，其余资产类（options/forex/crypto/futures/indices）一律 403。
- 数据类型：
  - `**minute_aggs_v1/YYYY/MM/YYYY-MM-DD.csv.gz`** — 分钟 bar，**主数据（频率杠杆）**。
  - `day_aggs_v1/…` — 日 bar（全量仅 0.9GB；同源日频，替代 legacy akshare 日频）。
  - `trades_v1` / `quotes_v1` — 逐笔成交 / NBBO 报价（微结构，量巨大，以后用，暂不下）。
- schema（aggregates）：`ticker, volume, open, close, high, low, window_start(纳秒 epoch), transactions`。
- **一文件 = 一交易日全市场所有 ticker**（~1 万只/天，**含全部 ETF**——ETF 在 SIP 里就是普通 ticker，所以"只订 US Stocks"就覆盖了我们整个跨资产层）。
- 下载器（`download_massive.py`）**纯 S3 镜像**（并发+断点续传+原子写），过滤到 universe / 重采样在 pipeline 做。
- **复权**：flat file aggregates 是 **as-traded（不复权）**——同 akshare 时代，拆股仍由管道自调；分红未调整（已知局限，raw 非总收益）。
- **train=serve**：研究用历史 flat files；**上线实盘要实时 → 需 Massive 实时档（Advanced $199）**。

## ⚠️ 滚动 10 年窗口（关键运营陷阱，曾导致"卡死"）

- 订阅是**滚动 10 年**：服务器 `LIST` 列出全历史 2003–2026（5726 文件 / 84GB），但**只有最近 10 年能真正下载**，起点 = **2016-06-14**（=今天减 10 年，**每天右移一天**）。
- 窗口外的 ~3200 文件**永久返回 403**。下载器若从 2003 跑，会卡在前面这堆下不了的文件上**疯狂重试退避，看起来"完全卡死"**；更糟会把 key 拖进真·限流惩罚，连合法文件也 403。
- **修复**：`download_massive.py --start-date 2016-06-14`（直接排除窗口外文件，不浪费重试）；稳妥并发（workers/rate ~6）。窗口内全量 = **2514 文件 / 45.3GB**。

## 历史深度的取舍（重要变化）

- akshare 日频曾给到 **1972/2004**（白送 87 股灾、互联网泡沫、08 GFC 等多轮 regime）。
- Massive 滚动窗口**只到 2016**——**用深 regime 历史换来了分钟分辨率**（×390 数据/天）。
- 对**日内验证 OK**：日内微结构/季节性比宏观 regime 平稳得多，不需要深历史（见 实验.md 频率讨论）。
- 深 regime 鲁棒性（08 式崩盘）= v2 的事，要时上 Massive Advanced（档案到 2003）。

**按数据类型汇总（关键）**：


| 类型                       | 体积          | 占比       | 我们要吗                    |
| ------------------------ | ----------- | -------- | ----------------------- |
| quotes（NBBO 逐笔报价）        | ~132 TB     | **95%**  | ❌ 做市商/HFT 数据，永不下        |
| trades（逐笔成交）             | ~4 TB       | 3%       | 选择性采样（微结构特征）            |
| **aggregates（分钟+日 bar）** | **~0.3 TB** | **0.2%** | ✅ **主力**（全资产分钟仅 ~284GB） |


总计 138.24 TB，但 **95% 是我们不交易的尺度（quotes）**——按"交易尺度匹配数据类型"自动砍掉，不是主观删。

**真正相关的全资产分钟 aggregates ≈ 284GB**（股票84 + 期权36 + 指数114 + 期货7 + 外汇35 + 加密7）。

**我们的取数逻辑**（机械、非人为先验）：

1. 按尺度：只取 aggregates（日内到数日），永不碰 quotes。
2. 按订阅：当前仅股票；期权（implied vol=新信息，最高价值新模态）是下一个值得加的单独订阅。
3. 按质量/流动性：管道机械过滤。
4. **混合比 + 采样是超参，验证驱动**（见 设计.md / 实验.md，不平摊、不靠人觉得谁重要）。
