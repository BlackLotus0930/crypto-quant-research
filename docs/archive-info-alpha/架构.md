## 一、端到端数据流(一图)

```
原始 OHLCV
  │  pipeline/clean.py   删错 → 身份切分 → 拆股还原 → 质量过滤
  │  归一(因果 scale-free, per-asset, 慢窗≈252日/1764bar)
  ▼
张量 [D, N, 5]                          # D=时间步, N=资产槽, 5=归一OHLCV
  n / mask / scale_ret / adj_close / dates / (日内多 tod)
  │  PanelDataset.__getitem__           # pipeline/dataset.py
  ▼
一个样本 = (anchor 日 a, 采样 Δt)
  x      [T, N, 5]   = n[a-T+1 : a+1]   # 因果窗口,只用 ≤a
  xmask  [T, N]
  y      [N]         = scale 归一的 a→a+Δt log 收益
  ymask  [N]         = 两端都观测的可预测 slot
  │  model/net.py  Encoder  +  model/head.py  QuantileHead
  ▼
pred [B, N, K=21]                        # 每个资产 21 个单调分位
  │  signal = pred.mean(-1)              # 收成一个方向数
  ▼
eval/backtest.py  riskeq sizing → 换手控制 → 扣成本 P&L → 净 Sharpe
```

关键事实:**输入只有 5 个归一 OHLCV 通道,无资产 ID、无静态特征、无基本面、无订单流。** 这是刻意的"any-variate / 无人类先验"设计(见 §三的取舍)。

## 二、模型本体

### 2.1 编码器 Encoder（[net.py](../model/net.py)）

```
embed: Linear(5 → d) × xmask  +  正弦时间位置编码(只在时间轴起作用)
for L 层 Block:
    资产轴注意力:  reshape [B*T, N, d]  → any-variate self-attn
                   key_padding mask 屏蔽空槽;无资产位置编码 = 置换等变
    时间轴注意力:  reshape [B*N, T, d]  → causal self-attn (is_causal)
    MLP:           LayerNorm → Linear(d→4d) → GELU → Linear(4d→d)
    （三个子层都 ×mask 残差;SDPA flash kernel,不物化 N×N）
readout: h[:, -1]                        # 只取 anchor 那一个时间步 → [B, N, d]
```

- **双轴是"可分解(factored/axial)"的**:每层先跨资产、再跨时间,**两轴分开做、不同时做**。
- 默认配置:`d=256, L=4, heads=4, T=64`(日频 E4 与日内 E7 同配置,为可比)。

### 2.2 分位头 QuantileHead（[head.py](../model/head.py)）

```
Δt → MLP → (γ, β)  做 FiLM 调制 h           # 用 Δt 这个标量条件化
base  = Linear(h)                            # [B,N,1]
inc   = softplus(Linear(h))                  # [B,N,K-1] > 0
out   = [base, base+cumsum(inc)]             # [B,N,K] 严格单调,分位不交叉
```

### 2.3 损失与训练（[train.py](../model/train.py)）

- **主损失 `ic_loss`**:每行(=一天横截面)的 Pearson IC,最大化 → `loss = -mean(corr)`。信号 = `pred.mean(-1)`。
- **辅损失 `pinball`**:K 个分位的 pinball(≈CRPS),做分布校准,权重 `λ=0.2`。
- AdamW + warmup+cosine + 梯度累积 + bf16 AMP;**按 val mean-IC 选最佳 epoch**;早停 patience=6。
- 监控(E7b 起):val_crps / val_IC / **IR=meanIC/stdIC** / rank-IC。

### 2.4 交易层（[backtest.py](../eval/backtest.py)）

- signal = 分布均值;**riskeq sizing**:`z-score(signal)/vol`(vol=因果 scale_ret),横截面 gross=1。依据:模型预测的是**归一(去波动)**收益,按 1/vol 定仓才把"归一空间 IC"变现成原始收益。
- 换手控制:部分 rebal + 不动带(band);**执行点(rebal+band)在 val 上按成本模型选**(`best_exec_at`,不在 holdout 上扫=防过拟合)。
- **年化按结算期数**:`metrics`/`ic.py` 用 `252×bars_per_day`(日内按 bar,日频回落 252)。修了"日内被 √252 低估 √7"的 bug。
- GO 判据:扣 2bps 净 Sharpe ≥ 0.5。

### 2.5 E7a 诊断结论（2026-06，决定下一步的关键事实）

- **真瓶颈在信号本身,不在交易策略**:`diag_strategy.py` 测出真实**毛 Sharpe 仅 +0.25**(√252),而 riskeq/signal/rank **三种 sizing 毛都 ~0.2** → 换 sizing 救不回来。IC 真、稳(年化 IR 3.86)但**幅度小(0.012)**,叠加**低有效广度**(名与名齐涨,公共因子吃广度,E5/E6 territory)→ 毛薄。
- **频率未质变**:修正年化后 E7a 净@2bps=+0.15(非 +0.06),仍远低 0.5;同 5bps 成本下日频 E4(+0.10)仍胜 60min(−0.21)。
- **唯一能增厚 IC 幅度的真杠杆 = scale**(想法.md 核心检验);机器层(E7b 的 tod/去季节化)只能边际帮忙。

### 2.6 E7b 改动（已就绪,本地验证通过,**未上卡**）

- **tod 输入通道**:`sin/cos(2π·slot/bars_per_day)` 作为第 6/7 通道(C 5→7),广播到所有 N。给模型"这根 bar 几点"的真坐标。模型侧仅 `Net(C=)` 串参,dataset 自动带通道,零侵入。
- **成交量 per-slot 去季节化**:`n_volume` 按 (ticker,slot) 估均值/std,除掉开收盘放量的 U 型(验证:跨槽 |均值| 0.366→0.050)。
- **scale_ret 保持 per-ticker 池化**:per-slot 会把隔夜(slot0 进/slot6 出)与日内 scale 错配→边界炸(slot6 normvol 2.56→5.35);且 ic_loss 横截面相关、scale 不变,池化无损。
- 产物:`intraday_tensor_60min_tod.npz`(C=7,mask 0.872,保留 E7a 的 `_60min.npz` 不动)。
