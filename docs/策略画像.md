# 策略画像:小资金套利 vs 大资金稳定

> 同一套大脑(`strategy.py`),两组参数 = 两个画像。**随资金规模切换**。
> 核心权衡:**集中肥尾薄币(每元收益肥、容量小、灵活)** ↔ **广撒厚币(每元收益薄、容量大、稳)**。

## 为什么有两个画像

边(funding 拥挤分散)在**极端 funding 的薄/新币上最肥**(肥尾,台账 E46/E48:净 ~2-4× 中位)。
- **小资金**($5-10k):薄肥尾币每个只能吃 $500-1000,但大资金吃不下 → **小资金的专属优势**。该**集中肥尾**。
- **大资金**(数十万+):薄币塞不下 → 必须**广撒在厚的流动币上**,牺牲每元收益换容量。

→ 不是哪个"更好",是**匹配你的资金规模**。

## 两个画像的参数

| 维度 | 小资金套利(当前默认) | 大资金稳定 | 谁是主要容量杠杆 |
|---|---|---|---|
| **流动阈值** `venues.route(min_qv)` | **$1M**(进薄肥尾) | **$20M+**(只厚币) | ★★★ 最大 |
| **venue 集** `venues.ACCESSIBLE` | **{hl, gate}**(2 所,够小资金宽度) | **{hl, okx, gate}**(3 所,加 OKX 增 venue/容量/宽度) | ★★★ |
| **倾斜** `tilt_pow` | 2(倾肥尾) | 1(线性广撒) | ★ 次要(被 cap/平滑稀释) |
| 每币帽 `cap` | 0.03/0.05 | 0.03/0.05(都低 cap 最优) | — 不分大小 |
| 杠杆 `leverage` | 低起步 1x | 同 | — |

venue 集都是**加拿大可达所**(HL/OKX/Gate,非 Binance/Bybit——后者已退出加拿大):
- 小资金:**HL+Gate** 够(HL+Gate 已给 49 币 @$1M,够小 size;少一个所=灰色暴露更小)。
- 大资金:**+OKX** 凑 3 所(更多 venue=更多跨所对+更多厚币宽度=更大容量;代价是多一个灰色所 OKX,大资金为容量值得)。

## 各画像的特性(实测/估)

| | 小资金套利 | 大资金稳定 |
|---|---|---|
| 每元收益(毛 funding) | 肥(肥尾 +14-40%) | 薄(中位 +8-13%) |
| 容量(边衰减前) | ~$100k-500k | 数百万+(受 venue 容量) |
| 稳定性 | 较集中(薄币、新币波动) | 更广更稳(厚币) |
| 适用资金 | $5k - ~$200k | ~$200k+ |
| 风险 | 薄币执行/基差/反转 | 低,但每元收益低 |

## 分层对冲 + 各层 funding + 没盖的尾部

把策略想成**洋葱**:最里两层产生 funding 收入,外面所有层都在保护这份收入。**funding 只在 ①② 两层产生。**

### ① carry 腿内对冲(delta 中性)
- **怎么对冲**:多 1 份现货 + 空 1 份永续(同币同所 Gate)。现货涨→永续也涨,多头赚≈空头亏 → **价格 delta≈0**。
- **残差风险**:现货 vs 永续**基差**(小,到期收敛)。
- **funding**:永续正 funding 时空方收钱;你空 perp → 每结算周期收 funding。**零借币**(perp 是衍生品)。
- **收多少**:正 funding 币 per-hour 累积 → **诚实 +4.7%/yr OOS**(2022 −4.6% 是 carry 最差年)。

### ② 跨所腿内对冲(delta 中性)
- **怎么对冲**:多 A 所永续 + 空 B 所永续(同币 HL/Gate)。两边一起动 → **价格 delta≈0**。
- **残差风险**:**跨所基差**——两所 mark 不同,尤其 HL 是 DEX(自有链上价/预言机),per-bar σ~8%(1.4×CEX)。这是跨所 Sharpe 比 carry 低(2.8 vs 9.9)的原因。
- **funding**:short 高 funding 所(收)+ long 低 funding 所(付)= **净收价差 |f_高−f_低|**。零借币。
- **收多少**:**诚实 +18%(Bin-Byb 代理)/ +24%(真实 HL 12mo)**(台账 E52)。

### ③ 截面分散(每币 cap)
每币权重封顶(carry 3%/跨所 5%);单币爆雷(基差爆裂/逼空/下架)最多伤 cap 比例本金。风控,无 funding。

### ④ 两腿分散(carry ⊥ 跨所,corr −0.16)
组合 = 50% carry + 50% 跨所,来源不同 → 一腿差另一腿补(**2022 carry −4.6% 时跨所 +16~24% 拉正组合**)。√N 红利:组合 Sharpe ≈ (S₁+S₂)/√2。**这是组合书的核心价值。**

### ⑤ 执行/风控叠加(代码里的保护)
- **流动过滤**(`route(min_qv)` + carry 现货腿过滤):排进不去/出不来的薄币 → 对冲执行风险。
- **风险调整路由**(λ=0.1 罚 HL):选所对 `spread − λ·基差风险`,偏干净 CEX → 对冲跨所基差。
- **no-trade band + EMA 平滑**:抑制抖动控换手成本。**杠杆帽**(1→2x):限回撤放大。

### 🔴 没盖的尾部(裸奔,实盘真正会栽的地方)

| 风险 | 状态 |
|---|---|
| **对手方/交易所**(Gate/HL 暴雷、冻结提现) | **没对冲,最大尾部**(小资金下 > 任何市场波动) |
| **强平时间错配**(空腿被暴拉,强平早于调仓) | 没建模 |
| **稳定币抵押脱锚**(USDT/USDC) | 没对冲 |
| **极端基差/脱钩尾部**(超历史样本的 delta 失效) | 没对冲;样本里也没出现过 → **回测 maxDD −3.6% 是低估** |

> **一句话**:5 层对冲(2 层产 funding + 3 层保护),但**对手方/强平/脱锚 3 个尾部一层没盖**。上线前的风控重点不是市场波动,是这 3 个。

## 诚实小字

- **tilt 本身差异小**(被 cap/平滑稀释,232 仓不变、maxDD 几乎不变);真正切容量的是**流动阈值 + venue 集**。
- **两画像都用加拿大可达所**(HL/OKX/Gate),不碰 Binance/Bybit(已退出加拿大)。大资金画像的容量上限 = 这 3 个可达所能吸收的厚币深度(比单 HL+Gate 大,但**仍低于大 CEX 理论容量**——这是加拿大可达性的真实天花板)。
- 资金成长路径:$5-10k 小资金(HL+Gate,低阈值,tilt2)→ 长大调高阈值 + 加 OKX 凑 3 所 + 倾斜回线性 → 大资金画像(HL+OKX+Gate,高阈值,tilt1)。
- 当前生产 = **小资金套利画像**(适配用户起步 $5-10k)。
